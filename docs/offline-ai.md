# 浏览器本地 AI 规划

这个项目后续可以加入“在线业务系统 + 小巧离线 AI 辅助”的能力。AI 模型建议按需加载，并缓存到浏览器本地，避免影响首屏和普通业务操作。

## 推荐能力

- 故障描述自动分类：根据用户输入推荐故障类型。
- 工单紧急程度判断：根据关键词、设备状态和客户描述给出优先级建议。
- 客户回访标签：自动标记满意、一般、投诉、需跟进。
- 维修记录语义搜索：用 embedding 在本地查找相似案例。
- 离线辅助：无网络时仍可做文本分析，联网后同步业务数据。

## 推荐技术顺序

1. `Transformers.js`
   - 适合小模型推理和 embedding。
   - 集成成本低，适合 Vue 前端直接接入。

2. `ONNX Runtime Web`
   - 适合自有 ONNX 模型。
   - 控制力更强，但工程复杂度更高。

3. `WebLLM`
   - 适合浏览器内本地聊天助手。
   - 依赖 WebGPU，模型体积较大，建议作为高级可选功能。

## 落地建议

- 模型不要放进首屏包，统一做懒加载。
- 模型文件放 CDN 或静态资源目录，结合 Service Worker 缓存。
- 推理结果可缓存在 IndexedDB，减少重复计算。
- 先做分类和检索，不急着做完整聊天机器人。
